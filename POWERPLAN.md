# POWERPLAN.md — 전원 콘센트 배치 현황

> 본 문서는 [`outlets.js`](outlets.js) 의 `OUTLETS` 배열에 정의된 모든 콘센트의 위치·구수·종류를 방별로 정리한 인덱스다.
> 좌표 단위: m. `xBR=7.8`, `xKit=5.1`, `xHall=10.5`, `zM1=2.7`, `zM2=4.2`, `zBath=4.8`, `zB=6.6`, `zLR2=3.5`, `WT=0.12` (DESIGN.md §2.2 참조).
> 모든 콘센트는 한국 220 V Type-F 형태 — 원형 리세스 컵 ⌀46 mm + 둥근 핀 홀 2 개 ⌀4.4 mm (19 mm 가로 간격). 2 구는 세로 배치 (위·아래), 그 외 (1/3/4 구) 는 가로 배치.
> 벽으로부터 돌출 거리: `CLEARANCE = 0.035 m` + `PLATE_T/2 = 0.0015 m` ≈ 37 mm.
> `face`: 'N'/'S'/'E'/'W' = 콘센트 plate 가 *향하는* 방향 (방 안쪽). N=+z, S=-z, E=+x, W=-x.
> `kind`: 'wet' = 욕실 방수형 (살짝 베이지 색).
>
> ⚠️ Public GitHub repo 미러 — 신상 식별 정보 추가 절대 금지 ([PRIVACY.md](./PRIVACY.md)).

## 표준 마운트 높이 — 3 단계 통일 (사용자 요청, 2026-05-08)

모든 OUTLETS 의 y 값을 다음 3 단계 중 하나로 통일. 각 콘센트는 가장 가까운 단계로 스냅.

| 단계 | y (m) | 용도 |
|---|---|---|
| **낮음 (low)** | **0.40** | 일반 콘센트 (거실/침실/현관/복도/창고/거실확장 인덴트 하부 등), 비데용, 냉장고 뒤, 침실 침대맡, 페어 하부 |
| **중간 (medium)** | **1.06** | 주방 작업대 / 세탁기·가전 / 세면대 위 / 서재 북측 4구 |
| **높음 (high)** | **1.80** | 주방 상부장 위 / 인덴트 상부 / 욕실 거울장 안쪽 / 발코니 방화문 옆 / 페어 상부 |

**스냅 적용 (CL 50318+)**:
- 0.30 / 0.70 / 0.40 → 0.40 (low)
- 1.061 / 1.10 / 1.20 → 1.06 (medium)
- 1.45 / 1.80 → 1.80 (high)

## PP (전원 계획) 모드 동작 요약

`2` 키로 토글 (이전 `1`, CL 50974 에서 변경). `1` 키 = **5/8 미팅 결정사항 모드** (`meetingmode.js`, CL 50995) 가 같은 시각 효과 (`_applyOutletView` 헬퍼) 공유 — PP 모드와 mutually exclusive (한 모드 켜면 다른 모드 자동 종료). 활성 시:
- 신발장(@FURN#50) 외 모든 가구 hide. 욕실 가구는 visible 유지 (거울수납장·변기·세면대·샤워파티션·휴지걸이·벽등).
- 캐비닛 도어 (주방 하부장·플랩 상부장·붙박이장·냉장고 등 — pivot 이 hide-target 가구 bbox 안) 도 함께 hide.
- 벽 도어 (방 사이 / 외부 / 신발장 / 영림 중문) 보존.
- 모든 콘센트 plate 둘레에 노란 BoxHelper 외곽선 영구 표시.
- 미니맵 배지: 벽 번호만 표시 (방·문·가구·창문 배지 숨김).
- 크로스헤어가 콘센트 가리키면: 라벨에 `콘센트 N: <라벨> (<구>구[, 방수]) • 높이 ## cm`.
- SHIFT 누르면 추가로 벽 번호 라벨 표시 (방·문·가구·창문 모두 무시).

## 데이터 / 빌더 위치

- 데이터: [`outlets.js`](outlets.js) 의 `OUTLETS` 배열 (현재 26 항목)

- 통계 헬퍼: `window._outletStats()` — total / totalGangs / byRoom / byKind 반환
- SHIFT-aim 검출: 같은 파일 `_outlets[]` 글로벌 레지스트리, [`minimap.js`](minimap.js) 의 `_getAimInfo()` 가 참조
- PP 모드: [`powerplan.js`](powerplan.js) — `setPowerPlanMode(on)` / `_initPowerPlanCache` / `_initOutletOutlines` / `_buildPpVisIdxs`
