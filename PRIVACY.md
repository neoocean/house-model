# PRIVACY.md — 본 repo 의 공개·비공개 정책

본 저장소는 **공개 (public) GitHub repo** 입니다. 모든 commit·이슈·코멘트가 누구에게나 보입니다.

거주자·시공자·금전 정보 같은 사적 컨텍스트는 본 저장소에 절대 포함되지 않도록 다음 정책을 따릅니다. **사람·LLM 모두 push 전 본 문서를 따라 검증해야 합니다.**

---

## 1. 절대 commit 하지 말 것

다음 항목은 어떤 파일(`.md` / `.html` / `.js` / 주석 / commit message / 이슈 / PR description) 에도 포함되어선 안 됩니다.

### 1.1. 거주자·물건 식별
- 단지명·동 이름·호수·아파트 브랜드명
- 도로명·지번·동/구/시 같은 주소 정보
- 평면도 출처 (예: 부동산 사이트 + 단지명 조합)
- 거주자·소유자·세대원 본명·SNS 핸들·연락처
- 입주 예정일·실거주 일자

### 1.2. 거래·계약
- 시공사·인테리어 디자이너·디자인 회사 이름
- 가구·가전 구매처 (오프라인 매장명 / 온라인 쇼핑 쇼룸 ID 등)
- 이사 업체·청소 업체·임대업자 이름
- 부동산 중개·법무·세무 업체 이름

### 1.3. 금전
- 견적 금액·계약 금액·계약금·중도금·잔금
- 지출 합계·예산
- 보증금·월세·관리비
- 가구·가전 단가·할인율·총 비용

### 1.4. 외부 첨부 파일
- 견적서·계약서·청구서·영수증 PDF 의 파일명·경로·내용
- 회의 녹음·사진·동영상 의 파일명·경로
- 위 자료를 보관하는 외부 디렉토리 경로 (예: `../todo-XXXX/...`)

### 1.5. 시스템 / 작업 환경 메타데이터
- 비공개 VCS 사용자명·서버·depot 경로 (예: `//username/...`)
- 로컬 작업 디렉토리 절대 경로 (홈 디렉토리·사용자명 노출)
- 비공개 워크스페이스명·세션 ID·티켓 정보

## 2. 본 repo 에 포함해도 되는 정보

- 익명화된 평면 치수 (예: 12 m × 6.6 m), 벽 위치, 방 구획 — 한국 표준 평면 다수가 비슷
- 일반 가구·가전 모델 번호 (LG DIOS, 데스커 등 — 카탈로그 공개값)
- 코드 동작·아키텍처·디자인 결정·시행착오 (기술 정보)
- 변경 이력 (Three.js 버전·CL 번호 등 — 비공개 VCS 의 *번호만*, 사용자명은 ✗)

## 3. 의심스러우면

다음 질문에 “Yes” 가 하나라도 있으면 commit 하지 말 것:

- 이 정보로 “집의 위치/소유자/계약 상대” 가 식별/추정 가능한가?
- 이 정보가 본인 또는 거래 상대방의 재정/사업 정보인가?
- 이 정보가 본 repo 외부에서만 의미를 가지는가 (외부 파일 경로 등)?

→ 비공개 작업 디렉토리(외부) 에만 보관.

## 4. push 전 자동 점검 (권장)

다음 패턴이 git diff 에 포함되어 있지 않은지 grep:

```bash
git diff --staged | grep -nE \
  '단지명_keyword|시공사_keyword|거래처_keyword|구체_금액_숫자|외부_파일_확장자' \
  && echo "⚠️ 위 패턴 발견 — 검토 후 진행"
```

(실제 키워드는 본인 환경에 맞게 채울 것 — 본 파일에는 키워드 자체를 적지 않음)

스크립트 예시:
```bash
#!/bin/bash
# push-check.sh — 자주 실수할 수 있는 키워드 체크
patterns=(
  "단지명_샘플"   # 본인 단지명
  "시공사명"
  "거래처명"
)
diff=$(git diff --cached)
fail=0
for p in "${patterns[@]}"; do
  if echo "$diff" | grep -q "$p"; then
    echo "⚠️ '$p' 발견 — commit 차단"
    fail=1
  fi
done
exit $fail
```

이 스크립트를 `.git/hooks/pre-commit` 에 두면 자동 검증. 패턴 자체가 이미 민감하므로 스크립트는 **개인 환경에만 보관, repo 에 commit 하지 말 것**.

## 5. 만약 이미 노출되었다면

GitHub 의 commit 이력은 **영구히 추적/캐시**됩니다. 단순 삭제로는 회수 불가. 다음 절차:

1. **회수 불가 가정** — 이미 외부 검색·아카이브에 인덱싱됐을 수 있음.
2. 영향 범위 파악 (어떤 정보가 노출됐는가).
3. GitHub: 해당 정보가 식별성 높으면 GitHub support 에 연락해 sensitive data removal 요청.
4. 필요 시 repo 를 private 으로 전환 + 새 repo 로 깨끗한 히스토리 push.
5. 노출된 정보가 거래처/시공사라면 사후 양해 협의.

예방이 회수보다 훨씬 쉽습니다 — 본 정책을 push 전 반드시 점검하세요.

## 6. LLM 에이전트 (Claude Code 등) 가 작업할 때

- [CLAUDE.md](./CLAUDE.md) / [AGENT.md](./AGENT.md) 상단 경고 박스를 항상 읽고 반영.
- 사용자가 외부 자료(계약서·견적·실측 사진 등) 를 보여줘도, 거기서 추출한 **치수·재질** 같은 비식별 기술 정보만 코드에 반영.
- 인명·단지명·계약자·금액·외부 파일명 등은 **요약·암시·간접 참조** 도 포함하지 말 것 (예: “이 단지는 보통 ~” 같은 표현도 식별 가능 시 금지).
- `MEMORY.md` / `DESIGN.md` 가 비식별 정보만 담고 있는지 commit 전 자체 검토.
